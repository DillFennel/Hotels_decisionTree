## Дерево решений для прогноза отмены брони

Проект решает задачу бинарной классификации: по параметрам бронирования нужно определить, будет ли отмена (`is_canceled = 1`) или нет (`is_canceled = 0`). Вся логика реализована в `Отели_Дерево_решений.ipynb` сделанном в Google Colab

### Данные
- Учебный датасет из `train_final.csv`, загружается с Google Drive. Взят с kaggle `https://www.kaggle.com/competitions/hotel-booking-demand-3`
- Всего 44 638 записей и 29 исходных признаков, включая данные о дате прибытия, количестве гостей, типах номеров, сегменте рынка и т.д.
- Целевая переменная — `is_canceled`.

### Подготовка данных
- Преобразование типа `children` к целому значению.
- Замена категории `Undefined` в `meal` на `SC`.
- Удаление неиспользуемого признака `reservation_status_date`.
- Очистка пробелов в категориальных признаках `hotel`, `market_segment`, `deposit_type`.
- One-Hot кодирование всех категориальных признаков (в результате размерность расширена до 229 признаков).
- Удаление выбросов в числовых столбцах (`stays_in_weekend_nights`, `stays_in_week_nights`, `previous_cancellations`, `previous_bookings_not_canceled`, `booking_changes`, `days_in_waiting_list`) с помощью правила 1.5 IQR.
- Разбиение на обучающую и тестовую выборки в пропорции 70/30 с `train_test_split` и `random_state=42`.
